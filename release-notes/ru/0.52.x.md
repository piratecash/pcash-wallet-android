## 🚀 Обновление версии 0.52.2
_Дата релиза: 17 марта 2026_

### ✨ Улучшения

- **Улучшен механизм синхронизации при переключении между кошельками и переходе в список транзакций**

- **Добавлено отображение оставшегося числа блоков для BEP-20 токенов**

- **Улучшен механизм отображения статуса кошелька в системных сообщениях**
  Сообщения «Пожалуйста, дождитесь окончания синхронизации» и «У вас ещё нет незавершенных или прошлых транзакций» теперь точнее отображают текущее состояние кошелька.

### 🐛 Исправления

- **Исправлена проблема наследования скрытия балансов при переходе с экрана ассета в обмен**

- **Исправлена ошибка создания ZEC-адаптера при переключении между кошельками**
  При переключении между кошельками через меню `Manage Wallets` адаптеры ZEC могли не создаваться, а все типы адресов (`Shielded`, `Transparent`, `Unified`) показывали ошибку `Another synchronizer is currently active`.
  В результате ZEC-кошелёк не синхронизировался после переключения аккаунтов.

- **Исправлен краш SQLiteDatabaseCorruptException в Zcash SDK на слабых устройствах**
  Приложение могло падать с ошибкой `SQLiteDatabaseCorruptException: database disk image is malformed` при работе с Zcash-кошельком.
  Сбой происходил внутри Zcash SDK при чтении транзакций из базы данных, когда повреждение БД не обрабатывалось и исключение пробрасывалось наверх как фатальный краш.

- **Исправлен краш Monero: invalid chunk state when deallocating address**

- **Исправлена проблема редактирования адреса в окне отправки**

- **Исправлена проблема прокрутки экрана после возврата из сканера в окне отправки**
  Если у пользователя была открыта клавиатура и он переходил в сканер, после возврата могла прокручиваться не вся часть экрана.

- **Исправлен краш при возврате на экран восстановления из бэкапа после фонового убийства процесса**

- **Исправлено перекрытие кнопки Restore системной панелью на экране Backup File**

## 🚀 Обновление версии 0.52.1
_Дата релиза: 12 марта 2026_

### ✨ Улучшения

- **Улучшен пользовательский опыт при отправке токенов TRX / JETTON**
  Добавлено отображение информации о недостаточном балансе токенов при отправке.

- **Добавлены дополнительные кнопки выбора суммы при отправке и обмене**
  Теперь доступны быстрые кнопки `25%`, `50%`, `75%` и `100%` для более удобного выбора суммы.

- **Улучшен механизм отображения статуса обмена**

- **Повышена скорость синхронизации для Bitcoin (BIP44)**

- **Доработан механизм отображения статуса стейкинга**
  Улучшено отображение процесса и состояния стейкинга.

- **Сокращена задержка обновления истории транзакций**
  Улучшен механизм обновления истории после переключения между кошельками.

### 🐛 Исправления

- **Исправлена проблема с позицией информации о процентах загрузки блокчейна на экране ассета**

- **Исправлено перекрытие ввода PIN-кода клавиатурой при блокировке**
  Если в момент блокировки клавиатура была активна, теперь она автоматически скрывается и больше не перекрывает ввод PIN-кода.

- **Исправлен краш при переключении между кошельками: TooManyRequestsException в ConnectivityManager**
  При многократном переключении между кошельками приложение могло падать с ошибкой `ConnectivityManager$TooManyRequestsException`.
  Причина была в превышении системного лимита Android на количество зарегистрированных `NetworkCallback` для одного приложения.

- **Исправлены проблемы падений при работе с Monero**

- **Исправлен краш при попытке подписать сообщение через WalletConnect**
  Краш возникал только в случае ошибки при обработке запроса (например, при разорванной или истёкшей сессии), когда приложение пыталось показать сообщение об ошибке.

- **Исправлена проблема свапа USDT TRC-20 на Polygon через Quickex**
  Ошибка возникала из-за того, что параметр finalNetworkFeeAmount возвращался как null.

- **Исправлена проблема отсутствия синхронизации USDT Jetton (Adapter Unavailable)**

- **Исправлен краш при восстановлении бэкапа (TransactionTooLargeException)**
  Приложение могло падать при восстановлении бэкапа из файла, если пользователь сворачивал приложение на экране ввода пароля после выбора файла бэкапа. Проблема чаще воспроизводилась на устройствах с ограниченными ресурсами и при большом размере файла бэкапа.

## 🚀 Обновление версии 0.52.0
_Дата релиза: 1 марта 2026_

### ✨ Улучшения

- **Обновлён внешний вид экрана ассета**
  Улучшено отображение экрана ассета для всех активов. Для Cosanta и PirateCash дополнительно отображаются статус стейкинга и невыплаченный остаток.

- **Добавлена информация о комиссии и количестве необходимых монет для обмена**

- **Рефакторинг экрана блокировки**
  Экран блокировки (ввод PIN-кода) перенесён из отдельной LockScreenActivity в overlay внутри MainActivity.
  Это позволяет сохранять состояние навигации при пересоздании процесса приложения.

- **Улучшена скорость загрузки приложения**
  Исправлена проблема, из-за которой приложение стало дольше загружаться и подтормаживать, начиная с версии 0.51.18-C6C92C8E7 (222).

### 🐛 Исправления

- **Исправлено отображение оставшегося числа блоков для синхронизации DASH**
  Ранее отображалось количество транзакций в сети вместо количества оставшихся блоков до завершения синхронизации.

- **Исправлен редкий краш при отправке TRC20 токенов**
  Приложение могло падать на экране подтверждения отправки.

- **Исправлен краш при открытии экрана подключения MiniApp на некоторых устройствах Huawei**

- **Исправлен краш при закрытии ZEC кошелька — SQLiteDatabase race condition**
  Приложение падало с ошибкой IllegalStateException: attempt to re-open an already-closed object: SQLiteDatabase при работе с ZEC кошельком.
  Краш происходил при закрытии синхронизатора, когда база данных закрывалась раньше, чем завершались активные запросы.

- **Исправлен краш после удаления кошелька и перехода на другой кошелёк**

- **Исправлены краши приложения при работе с Monero кошельком (libmonerujo.so)**

- **Исправлен краш при создании кошелька на устройствах с ограниченными ресурсами**
  При создании нового кошелька приложение падало с ExceptionInInitializerError.
  Краш происходил при инициализации класса WordList из библиотеки hd-wallet-kit-android — загружались все языковые списки слов одновременно, хотя для создания кошелька нужен только английский.

- **Исправлен краш UserNotAuthenticatedException при обновлении имени кошелька или флагов бэкапа**
  Приложение могло падать при переименовании кошелька или подтверждении бэкапа, если устройство не было разблокировано длительное время (7+ дней).

- **Исправлен ANR и краш при синхронизации ECASH из-за ошибки сериализации MerkleBlock**
  При синхронизации ECASH кошелька приложение зависало (ANR) и переставало отвечать.
  Потребление CPU достигало 578% в течение ~3.6 минут, после чего Android показывал диалог "Приложение не отвечает".
